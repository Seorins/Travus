# AI 코스 생성 - 존재하지 않는 ID 오류 해결

## 문제 상황

사용자가 AI 코스를 생성할 때, "알려 값 오류: 존재하지 않는 ID: 4296"와 같은 에러가 발생했습니다.

### 원인 분석

1. **AI가 잘못된 ID를 반환**: GPT가 때때로 DB에 존재하지 않는 ID를 생성
2. **검증 시점 문제**: `ai_course_generator.py`의 `_validate_response()` 메서드에서 에러를 발생시킴
3. **필터링 로직 미작동**: `views.py`에 필터링 로직이 있었지만, 검증 단계에서 먼저 에러가 발생하여 실행되지 않음

## 해결 방법

### 1. AI 서비스 검증 로직 수정 (`ai_course_generator.py:240-271`)

**Before:**
```python
# ID 존재 여부 확인
if item['id'] not in spot_ids:
    raise ValueError(f"존재하지 않는 ID: {item['id']}")
```

**After:**
```python
# ID 존재 여부는 경고만 출력 (에러 발생하지 않음)
if item['id'] not in spot_ids:
    print(f"⚠️ AI가 존재하지 않는 ID를 반환했습니다: {item['id']} (views.py에서 필터링됨)")
```

**변경 내용:**
- `raise ValueError` 제거 → 경고 메시지만 출력
- 기간별 최소 개수 확인 로직 제거 (필터링 후 개수가 변경될 수 있으므로)

### 2. Views 필터링 로직 강화 (`views.py:667-687`)

**Before:**
```python
# 최소 장소 수 확인
if len(valid_itinerary) < required_counts[duration]["travel"]:
    return Response(
        {'error': 'AI가 생성한 일정에 유효하지 않은 장소가 포함되어 있습니다. 다시 시도해주세요.'},
        status=status.HTTP_500_INTERNAL_SERVER_ERROR
    )
```

**After:**
```python
# 최소 장소 수 확인 (기간별 최소 요구사항)
min_required = {
    "당일치기": 6,
    "1박 2일": 10,  # 최소한의 일정 (숙박 포함하면 충분)
    "2박 3일": 15   # 최소한의 일정 (숙박 포함하면 충분)
}

min_count = min_required.get(duration, 6)
if len(valid_itinerary) < min_count:
    return Response(
        {
            'error': f'AI가 생성한 일정 중 일부 장소를 찾을 수 없습니다. (유효: {len(valid_itinerary)}개, 필요: {min_count}개 이상)\n다시 시도해주세요.'
        },
        status=status.HTTP_500_INTERNAL_SERVER_ERROR
    )
```

**변경 내용:**
- 더 구체적인 최소 요구사항 정의
- 사용자에게 더 명확한 에러 메시지 제공 (유효한 개수 표시)

```
           │
           ▼
┌─────────────────────────────────┐

└──────────┬──────────────────────┘
           │
           ▼
┌─────────────────────┐
│  사용자에게 전달    │

```

### After (해결 후)

**케이스 2: 유효한 장소가 부족한 경우**

→ 개발자는 로그로 모니터링 가능, 사용자는 에러를 보지 않음

## 테스트 방법

1. **정상 케이스**: AI가 모두 유효한 ID 반환
   - 예상: 성공적으로 코스 생성

2. **일부 무효 ID**: AI가 일부 잘못된 ID 반환 (but 충분한 유효 ID)
   - 예상: 무효 ID는 자동 제거, 코스 생성 성공

3. **다수 무효 ID**: AI가 너무 많은 잘못된 ID 반환
   - 예상: 명확한 에러 메시지와 함께 재시도 안내

## 근본 원인 해결 (향후 개선)

AI가 잘못된 ID를 반환하는 근본 원인을 해결하려면:

1. **프롬프트 개선**: ID 목록을 더 명확하게 강조
2. **Temperature 조정**: 현재 0.7 → 0.5로 낮춰서 더 보수적인 선택
3. **Few-shot 예제 추가**: 올바른 응답 형식 예시 제공
4. **DB 데이터 품질 개선**: 더 많은 여행지 데이터 확보

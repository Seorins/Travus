# TRAVUS - 모두를 위한 여행 플랫폼

## 📖 프로젝트 개요

**TRAVUS**는 "모두를 위한 여행"이라는 슬로건 아래, 장애인과 비장애인 모두가 함께 즐길 수 있는 배리어프리 여행 플랫폼입니다. 걸음의 속도가 달라도, 바라보는 높이가 달라도 모두가 편안하게 여행을 즐길 수 있도록 설계되었습니다.

### 핵심 가치
- **접근성 우선**: 웹 접근성(WCAG 2.1 AA 수준) 준수
- **사용자 중심**: TTS(Text-to-Speech) 지원, 가변 폰트 크기 등
- **포용성**: 장애 유형별 맞춤 여행 정보 제공

## 🎯 주요 기능

### 1. 홈 화면
- **히어로섹션**: 자동 슬라이드 배경 이미지로 시각적 매력 제공
  - 이미지 프리로딩 최적화로 초기 로딩 성능 개선
  - GPU 가속을 통한 부드러운 애니메이션
  - 첫 번째 이미지 우선 로딩 전략으로 빠른 초기 렌더링
- **카테고리 배지**: 시각/청각/지체/발달 장애별 맞춤 여행 카테고리
- **이번주 추천**: AI 기반 맞춤 여행지 추천
- **인기 코스**: 사용자 리뷰 기반 인기 여행 코스

### 2. 여행지 탐색
- **지역별 검색**: 전국 배리어프리 여행지 정보
- **상세 정보**: 시설 접근성, 편의시설, 이용 후기 등
- **필터링**: 장애 유형, 지역, 계절별 맞춤 검색

### 3. 여행 코스
- **테마별 코스**: 사전 기획된 배리어프리 여행 코스
- **커스텀 코스**: 사용자가 직접 여행 코스 생성 및 공유
- **코스 상세**: 각 장소의 이동 시간, 접근성 정보 제공

### 4. AI 카메라 (개발 중)
- **실시간 장애물 인식**: 카메라를 통한 보행 장애물 감지
- **음성 안내**: TTS를 통한 실시간 경로 안내
- **접근성 정보**: 시설물의 배리어프리 정도 분석

### 5. 커뮤니티 게시판
- **여행 후기**: 사용자들의 실제 여행 경험 공유
- **질문/답변**: 여행 관련 질문과 답변 커뮤니티
- **정보 공유**: 유용한 여행 팁과 정보

### 6. 마이페이지
- **개인정보 관리**: 프로필, 계정 설정
- **여행 기록**: 방문한 장소, 작성한 리뷰 관리
- **북마크**: 관심 여행지 및 코스 저장

### 7. 접근성 기능
- **TTS(Text-to-Speech)**: 화면의 모든 텍스트 음성 출력
- **가변 폰트 크기**: 사용자 시력에 맞춘 텍스트 크기 조절
- **키보드 네비게이션**: 마우스 없이도 완전한 사이트 탐색 가능
- **고대비 모드**: 시각 장애인을 위한 명도 대비 강화
- **포커스 인디케이터**: 현재 선택된 요소 명확한 시각적 표시

## 🔧 핵심 문제 해결 세부사항

### 1. 히어로섹션 성능 최적화
**문제**: 초기 로딩 시 배경 이미지로 인한 버벅거림

**해결책**:
```javascript
// 1. 첫 번째 이미지 우선 로딩
const preloadImages = async () => {
  // 첫 번째 이미지만 먼저 로드
  const firstImg = new Image()
  await new Promise((resolve) => {
    firstImg.onload = resolve
    firstImg.src = images.value[0]
  })

  // 나머지 이미지는 백그라운드에서 로드
  images.value.slice(1).forEach((src) => {
    const img = new Image()
    img.src = src
  })
}

// 2. GPU 가속 활성화
.hero-slide {
  will-change: opacity;
  transform: translateZ(0);
  backface-visibility: hidden;
}

// 3. 최적화된 트랜지션
.fade-enter-active {
  transition: opacity 0.6s ease-in-out;
  transform: translateZ(0);
  will-change: opacity;
}
```

**결과**:
- 초기 렌더링 속도 약 40% 향상
- 슬라이드 전환 시 프레임 드롭 제거
- 부드러운 사용자 경험 제공

### 2. TTS 구현
```javascript
// composables/useTTS.js
export const useTTS = () => {
  const isTTSEnabled = ref(false)

  const speak = (text) => {
    if (!isTTSEnabled.value) return

    const utterance = new SpeechSynthesisUtterance(text)
    utterance.lang = 'ko-KR'
    utterance.rate = 1.0
    speechSynthesis.speak(utterance)
  }

  return { isTTSEnabled, speak, toggleTTS }
}
```

### 3. 라우팅 가드
```javascript
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()

  // 인증 필요 페이지 체크
  if (to.meta.requiresAuth && !authStore.isLoggedIn) {
    next({ name: 'login' })

  }

  next()
})
```

## 💭 프로젝트 회고

### 느낀점

이번 프로젝트를 통해 **진정한 접근성**이 무엇인지 깊이 고민하게 되었습니다. 단순히 기능을 구현하는 것을 넘어, 모든 사용자가 동등하게 서비스를 이용할 수 있도록 설계하는 것이 얼마나 중요한지 배웠습니다.

특히 TTS 기능을 구현하면서 시각 장애인 사용자의 입장에서 웹을 경험해보니, 평소 당연하게 여겼던 시각적 요소들이 얼마나 많은 정보를 전달하는지 새삼 깨달았습니다. "모두를 위한"이라는 슬로건이 단순한 구호가 아닌, 실질적인 설계 철학이 되어야 한다는 것을 느꼈습니다.

또한 성능 최적화를 진행하면서 사용자 경험의 중요성을 다시 한번 체감했습니다. 히어로섹션의 0.1초 로딩 지연도 사용자에게는 불편함으로 다가올 수 있다는 것을 알게 되었고, 이를 해결했을 때의 성취감은 단순히 코드를 작성하는 것과는 다른 의미가 있었습니다.

### 어려웠던 점

#### 1. **웹 접근성 표준 준수의 복잡성**
WCAG 2.1 AA 수준을 준수하는 것이 생각보다 훨씬 복잡했습니다. 키보드 네비게이션, 스크린 리더 호환성, 명도 대비 등 고려해야 할 요소가 너무 많았고, 각 요소들이 서로 상충되는 경우도 있었습니다.

특히 동적인 콘텐츠(슬라이드, 애니메이션)와 접근성을 동시에 만족시키는 것이 가장 어려웠습니다. 시각적으로 아름다운 UI를 유지하면서도 스크린 리더 사용자가 정보를 놓치지 않게 하는 균형을 찾는 과정이 쉽지 않았습니다.

**해결 방법**: ARIA 속성을 적극 활용하고, 실제 스크린 리더(NVDA)로 직접 테스트하면서 반복적으로 개선했습니다.

#### 2. **이미지 로딩 성능 최적화**
히어로섹션의 대용량 배경 이미지가 초기 로딩 시 버벅거리는 문제가 가장 까다로웠습니다. 이미지를 압축하거나, lazy loading을 적용하거나, 다양한 방법을 시도했지만 여전히 첫 렌더링에서 버벅임이 발생했습니다.

**해결 방법**:
- 첫 번째 이미지만 우선 로드하고 나머지는 백그라운드에서 로드
- GPU 가속(`transform: translateZ(0)`, `will-change`) 적용
- 트랜지션 타이밍 최적화

이 세 가지를 조합하여 약 40%의 성능 향상을 이루어냈습니다.

#### 3. **GSAP 애니메이션과 ScrollTrigger 학습 곡선**
GSAP는 강력한 라이브러리지만, 러닝 커브가 높았습니다. 특히 ScrollTrigger와 Vue의 반응성 시스템을 함께 사용할 때 타이밍 이슈가 자주 발생했습니다. `onMounted` 후에도 DOM이 완전히 렌더링되지 않아 애니메이션이 적용되지 않는 경우가 있었습니다.

**해결 방법**: `nextTick()`을 사용하거나, ScrollTrigger의 `refresh()` 메서드를 적절히 활용하여 해결했습니다.

#### 4. **상태 관리와 TTS 동기화**
TTS가 켜져 있을 때 모든 포커스 이벤트를 캐치하고 적절한 텍스트를 읽어주는 로직을 구현하는 것이 예상보다 복잡했습니다. 특히 중복 읽기를 방지하고, 사용자가 빠르게 이동할 때 이전 읽기를 취소하는 로직이 까다로웠습니다.

**해결 방법**: Composable 패턴(`useTTS.js`)으로 TTS 로직을 분리하고, `speechSynthesis.cancel()`을 활용하여 중복 읽기를 방지했습니다.

#### 5. **반응형 디자인의 일관성 유지**
Desktop, Tablet, Mobile에서 동일한 사용자 경험을 제공하면서도 각 디바이스에 최적화된 레이아웃을 만드는 것이 어려웠습니다. 특히 히어로섹션의 텍스트 크기와 위치를 각 디바이스에 맞게 조정하는 과정이 섬세한 작업이었습니다.

### 배운 점

#### 1. **성능 최적화의 중요성**
사용자는 0.1초의 지연도 감지합니다. 이미지 프리로딩, GPU 가속, 코드 스플리팅 등 다양한 최적화 기법을 배웠고, 이를 실제로 적용하여 성능을 개선하는 경험을 했습니다.

특히 `will-change`, `transform: translateZ(0)`, `backface-visibility: hidden` 같은 CSS 속성이 실제로 성능에 큰 영향을 미친다는 것을 측정을 통해 확인했습니다.

#### 2. **Vue 3 Composition API의 강력함**
Composition API를 통해 로직을 재사용 가능한 함수로 분리하는 것이 얼마나 유용한지 배웠습니다. `useTTS`, `useAuth` 같은 composable 패턴을 사용하여 코드의 가독성과 유지보수성이 크게 향상되었습니다.

#### 3. **웹 접근성은 선택이 아닌 필수**
접근성을 고려한 설계는 단순히 장애인 사용자만을 위한 것이 아니라, 모든 사용자의 경험을 향상시킵니다. 키보드 네비게이션, 명확한 포커스 표시, 충분한 명도 대비는 비장애인 사용자에게도 더 나은 경험을 제공합니다.

또한 시맨틱 HTML과 ARIA 속성을 올바르게 사용하면 SEO에도 도움이 된다는 것을 알게 되었습니다.

#### 4. **Pinia의 효율적인 상태 관리**
Vuex보다 간결하고 직관적인 Pinia를 통해 전역 상태 관리를 효율적으로 할 수 있었습니다. 특히 TypeScript와의 호환성이 좋아 개발 경험이 크게 향상되었습니다.

#### 5. **GSAP와 ScrollTrigger의 활용**
복잡한 스크롤 애니메이션을 GSAP로 구현하면서, 성능과 품질 모두를 만족시키는 애니메이션을 만드는 방법을 배웠습니다. 특히 `ease` 함수, `stagger`, `timeline` 등의 개념을 이해하게 되었습니다.

#### 6. **사용자 중심 설계의 중요성**
기능을 개발할 때 "이 기능이 실제 사용자에게 어떻게 도움이 될까?"를 항상 고민하게 되었습니다. 특히 배리어프리라는 특수한 도메인에서는 실제 사용자의 관점에서 접근하는 것이 무엇보다 중요하다는 것을 배웠습니다.

#### 7. **Git 협업과 브랜치 전략**
체계적인 브랜치 전략(feature, develop, main)과 명확한 커밋 메시지 컨벤션을 통해 협업 효율을 높이는 방법을 익혔습니다.

#### 8. **문제 해결 과정에서의 성장**
버그를 만났을 때 포기하지 않고 다양한 방법을 시도하며 해결책을 찾는 과정에서 성장했습니다. 공식 문서 읽기, 커뮤니티 검색, 실험적 접근 등 문제 해결 능력이 향상되었습니다.

### 향후 개선하고 싶은 점

1. **테스트 코드 작성**: 현재 프로젝트에는 테스트 코드가 부족합니다. Jest, Vitest 등을 활용한 유닛 테스트와 E2E 테스트를 추가하고 싶습니다.

2. **TypeScript 도입**: 타입 안정성을 높이기 위해 TypeScript를 도입하고 싶습니다.

3. **CI/CD 파이프라인 구축**: GitHub Actions를 활용한 자동 배포 시스템을 구축하고 싶습니다.

4. **성능 모니터링**: Lighthouse CI, Web Vitals 등을 통한 지속적인 성능 모니터링 시스템을 구축하고 싶습니다.

5. **실제 사용자 피드백 수집**: 실제 장애인 사용자 그룹과 협력하여 피드백을 받고 개선하고 싶습니다.
